# Security Policy

## Ambito

Questo repository contiene principalmente documentazione, prompt, checklist, dataset didattici simulati e materiali di governance per Prometeus Marcelus, Bibliotheca Promethea e Promethea Economica.

Anche se il progetto non e un'applicazione software tradizionale, trattiamo con attenzione:

- dati personali;
- dati sanitari;
- dati finanziari;
- dati clienti;
- materiali premium o privati;
- istruzioni che potrebbero essere usate in modo rischioso;
- prompt in ambiti salute, legge, finanza, lavoro, privacy, sicurezza, minori ed emergenze.

## Segnalare un problema

Se trovi un problema di sicurezza, privacy o contenuto sensibile:

1. Non pubblicare dati personali o contenuti riservati in una issue pubblica.
2. Apri una issue usando il template `Privacy check` solo se puoi descrivere il problema senza dati sensibili.
3. Se il problema contiene dati riservati, contatta direttamente il maintainer prima di pubblicare dettagli.

## Cosa segnalare

Segnala:

- dati personali pubblicati per errore;
- dati sanitari, bancari, legali o clienti non anonimizzati;
- materiali premium o privati esposti pubblicamente;
- link a documenti non destinati alla pubblicazione;
- prompt che promettono diagnosi, consulenze legali definitive, guadagni certi o decisioni automatiche su persone;
- istruzioni potenzialmente dannose, illegali, manipolative o ingannevoli;
- credenziali, token, password o segreti.

## Cosa non inserire nelle segnalazioni pubbliche

Non inserire:

- nomi completi;
- indirizzi;
- numeri di telefono;
- codici fiscali;
- referti;
- contratti;
- coordinate bancarie;
- dati di minori;
- screenshot contenenti dati sensibili;
- token o credenziali.

## Tempi di risposta

Obiettivo ragionevole:

- prima revisione: entro 7 giorni;
- classificazione del rischio: entro 14 giorni;
- correzione o mitigazione: secondo gravita e disponibilita.

## Classificazione rischio

- Basso: refuso, link sbagliato, contenuto ambiguo ma non sensibile.
- Medio: contenuto poco prudente in aree sensibili.
- Alto: dati personali o contenuti privati pubblicati.
- Critico: credenziali, token, dati sanitari identificabili, dati bancari o rischio immediato per persone.

## Principio Promethea

I figli di Promethea sono separati ma interconnessi.

Ogni passaggio di contenuto tra Prometeus Marcelus, Promethea Economica e Bibliotheca Promethea deve essere prudente, sicuro e documentato.

Vedi anche [Promethea Governance](PROMETHEA_GOVERNANCE.md).
