# Standard Classificazione Marcelus

## Versione

Standard: Marcelus Classificazione 2.0 Elevata

## Obiettivo

Evitare disordine quando la biblioteca cresce oltre 2000, 5000 o 10000 prompt.

## Regole sui Nomi

- Mantenere numerazione iniziale.
- Usare underscore tra parole.
- Evitare caratteri speciali complessi.
- Non rinominare cartelle gia pubblicate senza migrazione documentata.
- Non creare sinonimi inutili.

## Regole sui File

Ogni nuova raccolta di prompt dovrebbe avere:

- `README.md`
- `prompts.md` oppure file tematici numerati
- breve descrizione dell'utilita
- guardrail quando il tema e sensibile
- indicazione FREE / PREMIUM / PRIVATO

## Regole di Qualita

Ogni prompt Marcelus deve contenere almeno:

1. ruolo;
2. contesto;
3. obiettivo;
4. vincoli;
5. formato di output;
6. tono;
7. self-check;
8. fonti o limiti quando necessari.

## Aree Sensibili

Per salute, sicurezza, legge, finanza, AML, privacy e politica:

- non inventare norme, dati o diagnosi;
- usare `[DA VERIFICARE]` quando manca una fonte;
- chiedere fonti ufficiali;
- distinguere informazione da consulenza professionale;
- inserire avviso di emergenza quando opportuno.

## Regola Anti-Duplicazione

Non copiare lo stesso prompt in piu cartelle. Usare invece:

- link;
- indice;
- riferimento secondario nei metadati;
- raccolta trasversale solo se serve davvero.

## Regola Premium

Un prompt diventa premium quando contiene almeno uno di questi elementi:

- processo completo;
- checklist professionale;
- schema decisionale;
- template pronto per cliente;
- monetizzazione;
- procedura ripetibile;
- alta specializzazione.
