# Prometeus Marcelus - Roadmap fase 2

## Stato attuale

Il progetto ha tre asset reali:

1. Repository GitHub pubblico `marcelus-v-v01`.
2. Biblioteca pubblica da 2000 prompt Marcelus.
3. Custom GPT pubblico Prometeus Marcelus, collegato al metodo e alla documentazione.

La fase di generazione massiva prompt puo considerarsi chiusa. Da ora il valore non cresce aggiungendo altri prompt, ma migliorando qualita, ricerca, test, casi d'uso e affidabilita dell'agente.

## Critica realistica

### Punti forti

- La quantita e importante: 2000 prompt creano percezione di progetto ampio.
- La classificazione per macroaree rende il progetto piu serio di una semplice lista.
- Prometeus Marcelus ha un posizionamento comprensibile: AI Prompt Architect in italiano.
- GitHub e LinkedIn stanno iniziando a raccontare lo stesso progetto.
- Esiste gia un test 10 utenti, quindi il progetto non e solo teorico.

### Punti deboli

- 2000 prompt senza una navigazione molto chiara possono intimidire.
- Alcuni file sono molto lunghi: servono indici, esempi e percorsi guidati.
- GitHub e ancora piu archivio che prodotto.
- Prometeus deve essere addestrato a scegliere, non solo a generare.
- Serve una pagina iniziale molto semplice: cosa faccio, dove clicco, cosa ottengo.
- Il profilo LinkedIn va completato con link e post, ma senza sembrare gonfiato.

## Direzione strategica

La prossima fase deve trasformare Prometeus Marcelus da biblioteca di prompt a sistema operativo leggero per creare prompt verificati.

Formula consigliata:

Prometeus Marcelus = Metodo + Biblioteca + GPT + Test + Portfolio pubblico.

## Addestramento Prometeus Marcelus

### Obiettivo

Prometeus non deve rispondere come un generatore generico. Deve comportarsi come un architetto del prompt:

1. capisce il problema;
2. classifica l'area;
3. fa poche domande utili;
4. produce un prompt finale;
5. indica rischi, limiti e fonti;
6. propone una versione breve e una versione avanzata;
7. evita promesse assolute in salute, legge, finanza e sicurezza.

### Dataset operativo

Creare una cartella `PROMETEUS_MARCELUS/training/` con:

- `casi_test.md`: 30 casi realistici;
- `risposte_ideali.md`: risposte modello;
- `errori_da_evitare.md`: risposte troppo generiche o rischiose;
- `rubrica_valutazione.md`: criteri da 1 a 5;
- `prompt_di_sistema_versionato.md`: istruzioni del Custom GPT aggiornate.

### Test minimo

Ogni nuova versione di Prometeus deve superare:

- 10 test cittadini;
- 5 test business;
- 5 test salute informativa;
- 5 test GitHub/AI;
- 5 test creativi.

Non creare subito tre organization. Rischio: confusione brand.

Ordine consigliato:

1. Restare su profilo personale finche il progetto matura.
2. Creare organization `Prometeus-Marcelus` solo quando esistono almeno 3 repository puliti.
3. Tenere `LUNE_X` separato come brand creativo.
4. Non creare organization per SUPERBIBLIOTECA: meglio repository o cartella principale.
