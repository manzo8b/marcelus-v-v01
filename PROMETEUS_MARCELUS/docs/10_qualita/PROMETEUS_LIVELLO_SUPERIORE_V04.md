# Prometeus Marcelus - Livello superiore v0.4

Questa fase trasforma Prometeus Marcelus da raccolta di materiali a sistema operativo leggero.

## Da biblioteca a metodo

Il progetto non deve essere valutato solo per il numero di prompt.

Il valore reale e:

```text
richiesta confusa -> classificazione -> rischio -> prompt migliore -> checklist -> verifica
```

## Tre pilastri

### 1. Agente

Prometeus Marcelus deve rispondere con metodo, non con teoria.

Ogni risposta deve contenere almeno:

- classificazione;
- rischio;
- prompt pronto;
- checklist;
- prudenza.

### 2. Bibliotheca Promethea

La biblioteca deve restare navigabile.

Ogni macroarea, nel tempo, deve avere:

- README;
- INDICE_PROMPT.md;
- link ai prompt esistenti;
- lista prompt mancanti;
- distinzione FREE / PREMIUM / PRIVATO.

### 3. Casi reali

La crescita deve avvenire dai problemi reali.

Meglio 30 casi trasformati bene che 500 prompt aggiunti senza prova.

## Posizionamento

Prometeus Marcelus non e:

- un oracolo;
- un consulente legale, medico o finanziario;
- un generatore infinito di prompt;
- una promessa di successo.

Prometeus Marcelus e:

- un architetto di prompt;
- un metodo di chiarimento;
- un filtro di prudenza;
- una scossa cognitiva controllata;
- un ponte tra utente non tecnico e uso piu responsabile dell'AI.

## Prossima release

La release v0.4 deve concentrarsi su:

1. Istruzioni Custom GPT V4.
2. Sistema qualita.
3. Playbook tester.
4. Selezione di 10 casi reali eccellenti.
5. Prima pagina pubblica semplice: "Usa Prometeus Marcelus".

## Cosa non fare

- Non creare altri grandi blocchi di prompt senza indice.
- Non allargare troppo le macroaree.
- Non vendere premium prima di avere esempi forti.
- Non pubblicare casi sensibili.
- Non comunicare il progetto come "2000 prompt" prima di spiegare il metodo.

## Frase guida

Prometeus Marcelus non promette magie. Aiuta a fare domande migliori, con piu metodo, prudenza e verificabilita.
