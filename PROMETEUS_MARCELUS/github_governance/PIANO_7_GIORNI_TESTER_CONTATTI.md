# Piano 7 giorni - tester e contatti GitHub

## Obiettivo

Portare Prometeus Marcelus da progetto ben documentato a progetto testato da persone reali.

Il traguardo dei prossimi 7 giorni non e aumentare il numero di prompt.

Il traguardo e ottenere:

- 10 persone raggiunte in modo corretto;
- 3 feedback concreti;
- 3 Issues operative;
- 1 Discussion pubblica;
- 1 piccolo report finale.

## Regola di lavoro

Poco, ma fatto bene.

Ogni azione deve lasciare una traccia utile nel repository:

- Issue;
- Discussion;
- nota di feedback;
- miglioramento del README;
- nuovo caso test;
- correzione di un limite.

## Giorno 1 - Preparare il punto di ingresso

Azioni:

1. Usare come link principale la release Starter Kit:
   `https://github.com/manzo8b/marcelus-v-v01/releases/tag/v0.5.0-starter-kit`
2. Verificare che il README mostri subito:
   - cosa e Prometeus;
   - link allo Starter Kit;
   - link a Start Here;
   - come lasciare feedback.
3. Aprire una Discussion pubblica:
   `Porta un caso reale: Prometeus prova a trasformarlo in un prompt migliore`

## Giorno 2 - Cercare 10 contatti buoni

Non cercare numeri grandi.

Cercare persone o repository coerenti con almeno uno di questi temi:

- prompt engineering;
- AI safety;
- educazione digitale;
- data analysis;
- compliance;
- risk management;
- produttivita personale;
- formazione;
- food and beverage;
- strumenti AI in italiano.

## Scheda contatto

Per ogni contatto annotare:

```text
Nome o repository:
Tema:
Perche e pertinente:
Link:
Messaggio inviato:
Risposta:
Feedback utile:
Issue collegata:
```

## Giorno 3 - Primo messaggio breve

Messaggio consigliato:

```text
Ciao, sto raccogliendo feedback su Prometeus Marcelus, uno Starter Kit italiano open source per trasformare richieste confuse in prompt piu chiari, prudenti e verificabili.

Se ti interessa AI pratica, prompt engineering o educazione digitale, mi sarebbe utile un feedback concreto:

https://github.com/manzo8b/marcelus-v-v01/releases/tag/v0.5.0-starter-kit
```

## Giorno 4 - Trasformare feedback in Issues

Ogni feedback utile deve diventare una Issue.

Formato minimo:

```text
Titolo:
[Feedback] Migliorare ...

Contesto:
...

Problema:
...

Proposta:
...

Area:
...

Rischio:
Basso / Medio / Alto

Priorita:
Bassa / Media / Alta
```

## Giorno 5 - Primo report pubblico

Scrivere un breve report:

- quante persone contattate;
- quante risposte;
- cosa non era chiaro;
- quale parte dello Starter Kit funziona;
- quale parte confonde;
- prossima correzione.

## Giorno 6 - Migliorare un solo punto

Scegliere un miglioramento piccolo.

Esempi:

- accorciare l'inizio del README;
- aggiungere un esempio prima/dopo;
- migliorare la pagina Start Here;
- aggiungere una FAQ;
- chiarire il rapporto tra Prometeus e Bibliotheca Promethea.

## Giorno 7 - Chiudere il ciclo

Azioni:

1. Aggiornare changelog.
2. Chiudere o commentare le Issues create.
3. Pubblicare un breve post LinkedIn o X.
4. Decidere se preparare `v0.5.1-feedback`.

## 3 Issues operative consigliate

### Issue 1

Titolo:

`[Tester] Raccogliere 10 feedback sullo Starter Kit`

Obiettivo:

Validare se una persona esterna capisce Prometeus senza spiegazione privata.

### Issue 2

Titolo:

`[README] Rendere il primo impatto piu chiaro in 30 secondi`

Obiettivo:

Ridurre densita e aumentare leggibilita per nuovi visitatori.

### Issue 3

Titolo:

`[Discussion] Raccogliere casi reali senza dati personali`

Obiettivo:

Aprire una porta pubblica per test, domande e casi concreti.

## Discussion pronta

Titolo:

`Porta un caso reale: Prometeus prova a trasformarlo in un prompt migliore`

Testo:

```text
Hai una richiesta scritta male, vaga o confusa?

Puoi incollarla qui senza dati personali.

Prometeus Marcelus la usera come esempio per:

- capire il problema;
- classificare l'area;
- individuare rischi;
- proporre domande utili;
- trasformarla in un prompt piu chiaro e verificabile.

Non inserire dati sensibili, sanitari identificabili, documenti privati, codici fiscali, indirizzi, informazioni bancarie o conversazioni private.

Obiettivo: imparare insieme a fare richieste migliori alle AI.
```

## Criterio di successo

Il piano funziona se almeno una persona dice:

`Adesso ho capito meglio come chiedere aiuto a una AI.`

Questo vale piu di 100 stelle senza feedback.
