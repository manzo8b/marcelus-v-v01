# Routine settimanale GitHub

Questa routine serve a far crescere Prometeus Marcelus e Promethea Economica senza appesantire il lavoro.

Durata consigliata: 20-40 minuti a settimana.

## Obiettivo

Tenere GitHub ordinato, leggibile e utile.

La routine non serve a fare tutto. Serve a scegliere poche azioni giuste.

## Prima regola

Ogni settimana scegliere massimo 3 priorita.

Se tutto sembra urgente, niente e davvero governabile.

## Passo 1 - Controllo rapido Actions

Andare su:

`Actions`

Verificare:

- Markdown quality;
- Security policy check;
- Dependabot.

Se un workflow e rosso, prima si corregge quello.

Se e verde, si procede.

## Passo 2 - Controllo Issues

Andare su:

`Issues`

Dividere mentalmente le issue in:

- problema reale;
- miglioramento documentazione;
- privacy/sicurezza;
- prompt da migliorare;
- task Promethea Economica;
- LinkedIn/tester.

Scegliere massimo 3 issue da lavorare.

## Passo 3 - Controllo Discussions

Andare su:

`Discussions`

Cercare:

- domande reali;
- feedback di tester;
- idee utili;
- confusione ricorrente;
- richieste che possono diventare prompt o casi test.

Se una discussione produce lavoro concreto, trasformarla in Issue.

## Passo 4 - Project Kanban

Andare su:

`Projects`

Spostare le card:

- da Inbox a Da valutare;
- da Da valutare a In lavorazione;
- da In revisione a Pubblicato;
- da Pubblicato ad Archivio se completate.

Non lasciare tutto in Inbox.

## Passo 5 - Aggiornare un file piccolo

Ogni settimana aggiornare almeno uno tra:

- changelog;
- README;
- Start Here;
- un caso reale;
- una guida Wiki;
- una scheda Promethea Economica;
- una issue chiusa con risultato.

Meglio un aggiornamento piccolo e chiaro che un grande blocco confuso.

## Passo 6 - Collegamento LinkedIn

Se durante la settimana nasce un risultato utile, preparare un post breve:

1. problema reale;
2. cosa abbiamo capito;
3. cosa e stato pubblicato su GitHub;
4. limite o prudenza;
5. invito a testare.

Non fare post solo per dire "abbiamo lavorato".

## Passo 7 - Promemoria finale

Alla fine di ogni sessione scrivere:

```text
Fatto:
- ...

A cosa serve:
- ...

File toccati:
- ...

Commit:
- ...

Da controllare:
- ...

Prossimo passo:
- ...
```

## Routine minima da 10 minuti

Quando hai poco tempo:

1. controlla Actions;
2. scegli 1 issue;
3. aggiorna 1 file;
4. fai commit/push;
5. scrivi promemoria.

## Routine ideale da 40 minuti

Quando hai tempo:

1. controlla Actions;
2. controlla Issues;
3. controlla Discussions;
4. aggiorna Project;
5. lavora 1 caso reale;
6. aggiorna changelog;
7. prepara un post LinkedIn;
8. fai commit/push;
9. scrivi promemoria.

## Cosa evitare

- Aprire troppe issue senza chiuderne.
- Creare documenti duplicati.
- Cambiare impostazioni GitHub ogni giorno.
- Toccare Danger Zone.
- Mischiare FREE, PREMIUM e PRIVATO.
- Pubblicare dati sensibili.
- Monetizzare prima di avere fiducia e casi utili.

## Regola Promethea

La crescita deve essere sostenibile.

Poco, chiaro, verificabile, utile.
