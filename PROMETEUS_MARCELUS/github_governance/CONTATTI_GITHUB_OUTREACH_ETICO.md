# Contatti GitHub e outreach etico

## Obiettivo

Far conoscere Prometeus Marcelus dentro GitHub senza spam, senza messaggi invasivi e senza cercare scorciatoie artificiali.

La strategia corretta e:

1. rendere il repository chiaro;
2. trovare comunita affini;
3. partecipare con contributi utili;
4. invitare solo quando il progetto e pertinente;
5. trasformare ogni feedback in miglioramento visibile.

## Cosa evitare

- Non scrivere lo stesso messaggio a tante persone.
- Non aprire Issues promozionali in repository altrui.
- Non usare GitHub come fosse una mailing list.
- Non chiedere stelle in modo aggressivo.
- Non promettere che Prometeus risolve problemi legali, medici o finanziari.
- Non contattare persone usando dati personali estratti o non contestuali.

### 3. Discussions pubbliche

Le Discussions sono migliori delle Issues per presentarsi.

Metodo:

- leggere prima il tono della comunita;
- rispondere a una domanda reale;
- condividere un esempio utile;
- citare Prometeus solo se pertinente.

Formula:

`Sto lavorando a un metodo italiano open source per trasformare richieste confuse in prompt piu chiari, prudenti e verificabili. Se puo essere utile al tema, ho pubblicato uno Starter Kit qui: [link].`

## Come usare Issues nel nostro repo

Creare Issues semplici, non troppo teoriche.

Prime categorie:

- testare un caso reale;
- migliorare README;
- migliorare una macroarea;
- verificare privacy e fonti;
- proporre un esempio prima/dopo;
- segnalare un prompt confuso.

## Come usare Discussions nel nostro repo

Discussion consigliata:

`Porta un caso reale: Prometeus prova a trasformarlo in un prompt migliore`

Testo:

```text
Hai una richiesta scritta male, vaga o confusa?

Puoi incollarla qui senza dati personali.

Prometeus Marcelus la usera come esempio per:

- capire il problema;
- classificare l'area;
- individuare rischi;
- proporre domande utili;
- trasformarla in un prompt piu chiaro e verificabile.

Non inserire dati sensibili, sanitari identificabili, documenti privati, codici fiscali, indirizzi o informazioni bancarie.
```

## Metodo 10 contatti

Non cercare 100 persone.

Cercare 10 contatti buoni.

Profilo ideale:

1. persona che pubblica o cura prompt;
2. formatore AI;
3. sviluppatore interessato a LLM;
4. esperto data analysis;
5. persona interessata a AI safety;
6. tutor o docente;
7. professionista HR;
8. operatore food and beverage digitale;
9. persona che scrive guide pratiche;
10. founder o freelance con progetto educativo.

Per ogni contatto:

- leggere 2 minuti il profilo;
- capire se Prometeus e pertinente;
- scrivere un messaggio personalizzato;
- chiedere un feedback piccolo;
- non insistere se non risponde.

## Indicatore di qualita

Un buon contatto non e chi mette una stella.

Un buon contatto e chi lascia:

- una critica;
- un caso reale;
- una domanda;
- una Issue;
- una proposta di miglioramento.
