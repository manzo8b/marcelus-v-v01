# Checklist giorno 1 - ricerca tester GitHub

## Prima di contattare persone

- [ ] Usare come link principale la release Starter Kit.
- [ ] Aprire il registro contatti tester.
- [ ] Cercare massimo 10 contatti, non di piu.
- [ ] Leggere il profilo o repository prima di scrivere.
- [ ] Scrivere solo se il progetto e pertinente.
- [ ] Personalizzare almeno una frase.
- [ ] Non chiedere stelle.
- [ ] Chiedere un feedback concreto.

## Primo obiettivo

Trovare 3 persone davvero pertinenti.

Non serve completare subito tutti i 10 contatti.

## Criterio di qualita

Un buon tester:

- capisce il problema;
- lascia una critica;
- porta un caso reale;
- segnala una confusione;
- suggerisce una correzione.

## Dopo ogni risposta

- [ ] Annotare nel registro.
- [ ] Capire se serve una Issue.
- [ ] Non discutere per difendere il progetto.
- [ ] Ringraziare.
- [ ] Trasformare il feedback in miglioramento.
