# Strategia GitHub - far conoscere Prometeus Marcelus

## Obiettivo

Far crescere Prometeus Marcelus dentro GitHub come progetto italiano serio, utile e verificabile sul prompt engineering operativo.

Il posizionamento consigliato e:

`AI Prompt Architect italiano per prompt verificati, sicuri e utili`

## 1. Prima impressione

Il repository deve far capire in meno di 30 secondi:

- cos'e Prometeus Marcelus;
- a chi serve;
- cosa contiene;
- come provarlo;
- come contribuire;
- cosa e pubblico e cosa resta privato o premium.

Azioni:

- mantenere aggiornato `README.md`;
- mettere in alto il link a `PROMETEUS_MARCELUS/START_HERE.md`;
- creare badge semplici: prompt, training, issues, discussions;
- aggiungere screenshot o schema del metodo;
- evitare linguaggio troppo pubblicitario.

## 2. Repository topics

Aggiungere topics coerenti:

- prompt-engineering
- italian-ai
- custom-gpt
- ai-prompts
- knowledge-base
- github-portfolio
- ai-safety
- productivity
- education
- risk-management

## 3. Discussions come porta di ingresso

Usare Discussions per:

- domande dei tester;
- idee nuovi prompt;
- proposte di macroaree;
- segnalazioni di errori;
- richieste di esempi.

Categorie consigliate:

- Idee prompt
- Test Prometeus
- Domande utenti
- Fonti e verifiche
- GitHub e metodo

## 4. Issues come motore di miglioramento

Le Issues devono essere semplici e leggibili.

Usare sempre:

- problema;
- area;
- impatto;
- azione proposta;
- FREE / PREMIUM / PRIVATO;
- rischio privacy o fonti.

Label consigliate:

- `good first issue`
- `training`
- `privacy`
- `documentation`
- `prompt-library`
- `custom-gpt`
- `superbiblioteca`
- `linkedin`
- `fonti`

## 5. Project Kanban

Mantenere il Project `Prometeus Marcelus - Fase 2`.

Colonne consigliate:

- Inbox
- Da valutare
- In lavorazione
- In revisione
- Pubblicato
- Da collegare a LinkedIn
- Archivio

Routine:

- venerdi: revisione Issues;
- domenica: scelta priorita;
- lunedi: pubblicazione avanzamento.

## 6. Release pubbliche

Creare release semplici:

- `v0.1 - Biblioteca iniziale`
- `v0.2 - 2000 prompt`
- `v0.3 - Prometeus Marcelus V2`
- `v0.4 - Test pubblici e tester`

Ogni release deve spiegare:

- cosa cambia;
- per chi e utile;
- limiti;
- prossimi passi.

## 7. GitHub Pages

Creare una pagina pubblica semplice:

- cos'e Prometeus Marcelus;
- link al Custom GPT;
- link alla SUPERBIBLIOTECA;
- esempi di prompt;
- come diventare tester;
- avviso su sicurezza, privacy e fonti.

La pagina deve essere concreta, non una landing page gonfiata.

## 8. Collaborazioni

Target iniziali:

- formatori;
- tutor;
- piccoli imprenditori;
- studenti universitari;
- professionisti HR;
- operatori food and beverage;
- persone interessate ad AI pratica in italiano.

Metodo:

- invitare 10 tester;
- chiedere 1 caso reale per persona;
- trasformare feedback in Issues;
- pubblicare aggiornamenti su LinkedIn;
- chiudere il ciclo con report GitHub.

## 9. Contenuti LinkedIn collegati a GitHub

Formato post consigliato:

1. problema reale;
2. cosa ha fatto Prometeus;
3. cosa e stato pubblicato su GitHub;
4. cosa manca;
5. invito a testare.

Esempio:

`Sto costruendo Prometeus Marcelus, un Custom GPT italiano per creare prompt piu utili, sicuri e verificabili. Ora il progetto ha 2000 prompt, una SUPERBIBLIOTECA, test reali e Issues pubbliche su GitHub. Cerco 10 tester con casi concreti.`

## 10. Regola strategica

Non puntare subito alla quantita.

Da ora il valore cresce con:

- qualita dei test;
- chiarezza della documentazione;
- utilita reale;
- fiducia;
- sicurezza;
- esempi concreti;
- capacita di trasformare feedback in miglioramenti visibili.

3. Aprire una Discussion per raccogliere casi tester.
4. Pubblicare un post LinkedIn.
