# Shortlist tester GitHub - 2026-07-07

## Obiettivo

Creare una prima lista di luoghi GitHub dove cercare tester per Prometeus Marcelus.

Questa non e una lista di persone da contattare in massa.

E una mappa di topic, repository e comunita da valutare con prudenza.

## Regola

Prima si osserva.

Poi si valuta se il progetto e pertinente.

Solo dopo si scrive un messaggio breve, umano e non promozionale.

## Prime 3 azioni consigliate

1. Aprire `prompt-engineering`, `ai-safety` e `education-ai`.
2. Cercare solo repository con:
   - README chiaro;
   - attivita recente;
   - Issues o Discussions aperte;
   - tema coerente con prompt, AI pratica, sicurezza o educazione.
3. Inserire massimo 3 candidati nel registro:
   `PROMETEUS_MARCELUS/github_governance/REGISTRO_CONTATTI_TESTER_GITHUB.md`

## Quando non scrivere

Non scrivere se:

- il repository e inattivo da anni;
- il tema non c'entra;
- non ci sono Discussions o canali adatti;
- il messaggio sembrerebbe promozionale;
- il progetto tratta dati sensibili o casi personali;
- non hai letto almeno README e ultime Issues.

## Criterio di successo

Il primo successo non e una stella.

Il primo successo e trovare una persona che dica:

`Ho capito cosa fa Prometeus Marcelus, ma migliorerei questo punto...`
